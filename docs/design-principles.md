# Axiom Design Principles

## 1) Dark-First Visual System
- Use a near-black base (`#050505`) with restrained glow accents.
- Keep contrast high for readability, especially headlines and CTAs.
- Avoid color clutter; green/teal glows should carry hierarchy.

## 2) Clarity Over Decoration
- Every section must answer one user question quickly:
  - What is Axiom?
  - How does it work?
  - What services are provided?
  - What action should I take next?
- Prefer short, concrete copy over abstract marketing language.

## 3) Intentional Motion
- Motion should communicate state (hover, active, loading), not distract.
- Keep animation subtle and slow for premium SaaS tone.
- Use glow pulse selectively on critical UI only.

## 4) Glassmorphism With Discipline
- Glass effects should support grouping and depth, not overpower content.
- Keep blur levels consistent across cards and panels.
- Maintain visible card boundaries with soft borders for accessibility.

## 5) Systematic Composition
- Build sections with reusable visual patterns and spacing scales.
- Preserve consistent rhythm across hero, process, and services blocks.
- Design mobile first: no lost information, no broken hierarchy.

## 6) Brand Voice in UI
- Axiom should feel precise, modern, and operationally reliable.
- Copy and UI should signal outcomes: automation, speed, continuity.
- Avoid gimmicky metaphors that conflict with trust and performance.
